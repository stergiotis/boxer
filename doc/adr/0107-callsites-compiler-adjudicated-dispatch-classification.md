# ADR-0107: callsites ‚Äî compiler-adjudicated dispatch classification

## Status

Accepted ‚Äî 2026-07-05 (reviewed by @spx). Implemented in
`public/gov/callsites` (marker-driven test suite against a hermetic fixture
module; adjudication verified against the pinned toolchain).

## Context

`public/gov/callsites` exists to answer a governance question: **how much of
our code dispatches dynamically, and where do generic instantiations degrade
into dictionary-mediated calls?** The question is worth asking here ‚Äî leeway
and caching sit on hot paths where the difference between a stenciled generic
call and an interface-carrying one is measurable ‚Äî and no mainstream Go linter
covers it.

The first implementation (`e88ef8ce`) was a purely syntactic classifier over
`go/types`. An adversarial review (2026-07-05, findings reproduced by
executing the package against fixture modules) found it untrustworthy on every
axis it reports:

- **Wrong numbers, with systematic bias.** Conversions through composite type
  literals (`[]byte(x)`, `(*S)(nil)`) and immediately-invoked func literals
  counted as *dynamic* calls; methods on `*G[int]` (pointer receiver ‚Äî the
  common case for generic containers) counted as *monomorphic*; alias type
  arguments (`types.Alias`, materialized since Go 1.23) classified as
  `SubtypeNone`; same-module sibling packages counted as *3rdParty*.
- **Dead configuration.** The `Pattern` field and the CLI `--pattern` flag were
  never read; the analyzer always scanned the process working directory. Build
  tags were never wired, and the forced `-tags=` silently *hollowed out*
  tag-gated packages (a tag-gated package under `public/identity` surveyed as
  one call site) ‚Äî silent partial coverage in the exact repository the tool
  serves.
- **Broken streaming contract.** Terminating the `iter.Seq2` consumer early
  panicked (`range function continued iteration‚Ä¶`) because the `ast.Inspect`
  walk could not stop.
- **A cost model the compiler contradicts.** The subtype enum hard-coded
  Go 1.18-era lore ("struct/map/chan/array type args are shape/dictionary
  cases; `[]int` vs `[]string` differ"). Empirically (symbol dump below), the
  current toolchain stencils all of those per layout; only pointers collapse
  into a shared shape.
- **No oracle, no consumer, no live tests.** Two of three tests were
  `t.Skip`-disabled; the third asserted inside a match loop that never matched.
  Nothing in `scripts/ci/lint.sh` invokes the tool.

The compiler itself reports the facts the tool tried to infer: with
`-gcflags=-m` it prints per-position `devirtualizing i.M to S` and
`inlining call to F[go.shape.int]` lines (relative path, line, **column of the
call's opening parenthesis**), for library and main packages alike, on warm
caches too (verified on the pinned toolchain, go 1.26.4). This ADR rebuilds the
package around that ground truth.

## Decision

Rebuild `callsites` as a two-layer tool: a corrected static classifier that
states *what the code says*, joined against compiler diagnostics that state
*what the toolchain did*. Sub-decisions:

### ¬ßSD1 ‚Äî Two layers: classify statically, adjudicate with the compiler

The static layer classifies every `ast.CallExpr` from `go/types` facts alone ‚Äî
deterministic, position-canonical, toolchain-independent. The adjudication
layer (on by default in the CLI, optional in the API) runs
`go build -gcflags=-m` over the same patterns/tags, parses the
`devirtualizing ‚Ä¶` and `inlining call to ‚Ä¶` lines, and joins them onto call
sites by `(file, line, column-of-lparen)`. Adjudication **annotates** ‚Äî it
never overrides the static classification. A dynamic call the compiler
devirtualized stays `DynamicPolymorphic` and additionally carries
`Compiler.Devirtualized = true`; disagreement between the layers is signal
(the tool's raison d'√™tre), not error.

Definition-position lines (inlining decisions inside instantiated generic
bodies) and `<autogenerated>` positions do not correspond to scanned call
sites and are dropped by the join.

### ¬ßSD2 ‚Äî Call-site schema: position-canonical, provenance-explicit

`CallSite` v2:

- **Position** is `fset.Position(call.Lparen)` ‚Äî file, line, column of the
  opening parenthesis. This matches the compiler's diagnostic positions
  exactly (verified empirically), making the ¬ßSD1 join a strict key equality,
  and disambiguates chained calls on one line.
- **`Func`** is the qualified callee (`types.Func.FullName()` where resolvable:
  `(*pkg.G[T]).Set`, not a bare `Set`).
- **`CallTypeE`** gains honest categories instead of forcing everything into
  three: `Unknown` (zero value ‚Äî unresolvable, never fabricated), `Monomorphic`
  (direct static call, including immediately-invoked func literals),
  `StaticPolymorphic` (generic instantiation or method on an instantiated /
  type-parameter receiver), `DynamicPolymorphic` (interface dispatch or call
  through a func value), `Conversion` (type conversion ‚Äî not a call at all),
  `Builtin`.
- **Type-argument provenance is explicit**: `TypeArgs` carries the callee's own
  instantiation (generic function call), `RecvTypeArgs` the receiver's
  (method on `G[int]` / on a type parameter). The two are distinct fields, not
  one overloaded slice.
- **`Compiler CompilerDecision`** (`Checked`, `Devirtualized`, `InlinedCall`)
  carries the ¬ßSD1 adjudication; extension point for ¬ßSD8.

### ¬ßSD3 ‚Äî Shape cost model measured from the toolchain, not folklore

Each type argument is classified on one axis that the compiler's gcshape
behaviour actually exhibits, `ShapeClassE`:

| class | meaning | cost |
| --- | --- | --- |
| `ShapeClassStenciled` | own instantiation per layout (basics, strings, structs, arrays, slices, maps, chans, funcs) | ~monomorphic |
| `ShapeClassPointer` | all pointer types collapse into one `go.shape.*uint8` instantiation | dictionary-mediated; devirtualization lost |
| `ShapeClassInterface` | interface type argument | dictionary **plus** interface indirection ‚Äî the documented worst case |
| `ShapeClassTypeParam` | unresolved `T` passed through inside a generic body | resolved per outer instantiation; dictionary-mediated |

Evidence, reproducible against the pinned toolchain:

```sh
# a generic identity G[T] instantiated with two same-layout structs, one
# different-layout struct, []int, []string, map, chan, int, string, *A, *B:
go build -o /tmp/shapes . && go tool nm /tmp/shapes | grep -oE 'main\.G\[.*\]' | sort -u
```

yields distinct `go.shape` symbols for `[]int` vs `[]string`, for `map`,
`chan`, `[4]uint8`, and per struct *layout* (the two same-layout structs
merge), while `*A` and `*B` share the single `go.shape.*uint8` ‚Äî pointers are
the only kind-level collapse. The v1 enum's twelve syntactic subtypes and its
`SliceBasic`/`SliceGeneric` split encode distinctions the compiler does not
make and are dropped. The *descriptive* detail lives in a rendered type string
(`TypeArgInfo{Shape ShapeClassE, Type string}`) instead of a second enum.

Each `TypeArgInfo` classifies the argument's top-level shape only (after
`types.Unalias`, through named types' underlying); pointer-collapse *inside* a
composite (`[]*T` is its own slice shape with a collapsed element) is out of
scope for the per-argument verdict.

### ¬ßSD4 ‚Äî Origin is module-based

`OriginE`: `Local` = callee's package path equals, or is a segment-prefixed
child of, the scanned root's module path; `StdLib` = first path segment
carries no dot (plus universe/builtin callees); `3rdParty` = the rest;
`Unknown` = zero value. The v1 rule (`Local` ‚áî same *package*, dot anywhere ‚áí
external) made every cross-package call inside this repository "3rdParty".
Nested modules under one repository are accepted as approximation and
documented.

### ¬ßSD5 ‚Äî Loading is explicit; partial coverage is an error

`AnalyzerService` takes `Patterns []string`, `Dir string`, `BuildTags`,
`BuildFlags`, `IncludeTests` ‚Äî all honored (v1's `Pattern` was dead). `-tags`
is passed to `packages.Load` only when tags are actually supplied, mirroring
`codelint`'s loader. A package that fails to load terminates the run with an
error naming the package ‚Äî the tool must never silently survey a hollowed-out
tree (the v1 failure mode).

Constraint-excluded files are the second face of that failure mode: a package
whose tag-gated files are cleanly excluded loads without error, so fail-fast
cannot see the gap. Coverage is therefore reported explicitly ‚Äî an optional
`OnLoadStats` hook receives `LoadStats{Packages, IgnoredFiles}` (from
`packages.NeedFiles`/`IgnoredFiles`, test files not counted), and the CLI
logs the counts and warns when `IgnoredFiles > 0`, pointing at `--tags`.

### ¬ßSD6 ‚Äî Streaming contract that survives early termination

`All(ctx) iter.Seq2[CallSite, error]` (stdlib iterator naming, per codelint
CS010) keeps the streaming API shape, with the walk guarded by a stop flag so
a consumer `break` ends the sequence instead of panicking, and with `ctx`
cancellation checked between packages. Analysis is infallible per node (no
fabricated fallbacks ‚Äî unresolvable identifiers yield `CallTypeUnknown`); the
error slot of the sequence carries load/adjudication failures only.

### ¬ßSD7 ‚Äî CLI shaped like the sibling gov tools

`gov callsites [flags] [pattern ...]` (default `./...`): `--tags` (comma
list), `--adjudicate` (default true), `--fail-on <rule>` (repeatable). The one
rule shipped now is `interface-type-arg` ‚Äî any scanned call site passing an
interface as a type argument (the unambiguous DON'T from the sources in
`EXPLANATION.md`) ‚Äî exiting non-zero when matched, so `lint.sh` can gate on it
once the tool has soaked. A zerolog summary line (counts per call type, per
shape class, adjudication hit counts) closes each run, like `codelint`'s
completion line.

### ¬ßSD8 ‚Äî Deferred, with seams in place

- **Possible-target counts for dynamic calls** (x/tools `callgraph/vta`):
  answers "how open is this seam", complements ¬ßSD1's "what did the compiler
  do". Extension lands as fields on `CompilerDecision`'s sibling; no schema
  churn expected.
- **PGO join**: profile-driven devirtualization verdicts need representative
  profiles that CI does not have; revisit when profiles exist.
- **keelson/ClickHouse sink** (ADR-0094 introspection tables) and
  **`lint.sh` wiring**: mechanical once the survey output has soaked locally.

## Design space (QOC)

**Question:** what mechanism should a dispatch-governance survey trust?

| Criterion \ Option | v1 syntactic-only | **static + `-m` adjudication (chosen)** | SSA callgraph (VTA) core | PGO-profile core |
| --- | --- | --- | --- | --- |
| Matches toolchain truth | no ‚Äî measured divergences | yes ‚Äî compiler's own lines | partially ‚Äî feasibility, not decisions | yes, but only where profiled |
| Maintenance surface | small but rots silently | small; format drift caught by pinned-toolchain regression test | large (SSA build, whole-program) | large (profile pipeline) |
| Runtime cost | one `packages.Load` | + one cached `go build` | SSA of full program per run | needs instrumented runs first |
| Answers "where do generics degrade" | claims to; wrong buckets | yes (¬ßSD3 + shape strings in `-m` output) | no (callgraph ‚âÝ shapes) | indirectly |

## Alternatives

- **Keep the v1 syntactic classifier** ‚Äî killed: eight reproduced
  misclassification classes (Context above); with no oracle attached, every
  future toolchain change degrades it silently again.
- **SSA callgraph (CHA/RTA/VTA) as the core** ‚Äî killed as the *core*: it
  computes which targets a dynamic call *could* reach, not what the compiler
  *did*; it is the heaviest dependency in the space; and it says nothing about
  generic shapes. Retained as the ¬ßSD8 extension for seam-openness.
- **Compiler `-json` optimization-detail channel instead of `-m` text** ‚Äî
  killed empirically: on go 1.26.4 the schema-versioned JSON stream carries
  `canInlineFunction`/`cannotInlineFunction`/`leak` only ‚Äî no per-callsite
  inlining records and no devirtualization records (a devirtualized call
  produced `-m` text but no JSON diagnostic). The two `-m` line forms consumed
  here are the same ones gcassert-class tools have parsed stably for years.
- **gcassert itself (per-site source directives)** ‚Äî different tool shape:
  gcassert asserts hand-annotated expectations at chosen sites; this tool
  surveys every site without annotation. Related work, not a substitute.
- **PGO-first** ‚Äî killed for now: no representative profiles in CI; runtime
  hotness is a different axis than static dispatch shape (¬ßSD8).

## Consequences

- Exported API of the package changes incompatibly (enum values, `CallSite`
  fields, `Pattern` ‚Üí `Patterns`/`Dir`, `Run` ‚Üí `All`). Accepted: nothing in
  this repository invokes `gov callsites`, and the dead `Pattern` field made
  correct external use impossible ‚Äî any external caller that set it was
  silently scanning its process working directory, so migration to `Dir` is
  a bug fix for them, not just churn.
- The adjudication layer couples to the *human-readable* `-m` diagnostics.
  Contained: the toolchain is pinned in `go.mod`; a regression test asserts
  the two line forms still parse and still join (a devirtualized fixture call
  must come back `Devirtualized`); parse misses degrade to
  `Compiler.Checked = false`, never to wrong static classifications.
- Adjudication adds one `go build` per run (warm-cache safe, verified).
  `--adjudicate=false` keeps the fast path.
- The ¬ßSD3 shape table is itself toolchain-versioned knowledge. It is recorded
  here with its measurement procedure, not as timeless truth; re-run the
  `nm` probe when the toolchain moves and update via a dated `## Update`.
- Test fixtures live as a real module under `testdata/` (std-only imports plus
  a `replace`d local dep for the 3rdParty axis), so classification tests run
  hermetically and offline; the disabled v1 tests are replaced by live ones.

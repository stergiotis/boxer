# Changelog: 2026-07-02 – 2026-07-16

Covers 338 commits over a two-week window. The window is a date range, not a
release boundary: it contains [`v0.0.12`](https://github.com/stergiotis/boxer/releases/tag/v0.0.12)
(2026-07-09) and [`v0.0.13`](https://github.com/stergiotis/boxer/releases/tag/v0.0.13)
(2026-07-12), and ends with 77 commits after `v0.0.13` that are not yet tagged.
Twenty-five ADR numbers were opened in the window (ADR-0100 through ADR-0125;
0119 was never used). Twenty-three survive: ADR-0110 was folded into ADR-0113
and ADR-0120 was withdrawn. Eight of the survivors are still `proposed`, and are
marked as such below.

This summary was drafted from the commit range and the ADR bodies. Where an ADR
is the authority on a decision, follow the link rather than this entry.

## New: recordstore

[`public/storage/recordstore`](../../public/storage/recordstore) did not exist
before this window. It is a generated store over
[`leeway`](../../public/semistructured/leeway) records in ClickHouse, sitting on
the [`caching`](../../public/caching) layer, and was built out over four slices
([ADR-0100](../adr/0100-recordstore-generated-leeway-clickhouse-store.md),
accepted):

- **S1** — the generated store over leeway and caching (`255fae2f`).
- **S2** — `Scan` verbs, cache hardening, presence-gated `ReadRow` decode
  (`f84a65a3`, `3b21dd49`).
- **S3** — `pushoutstore`, a `StorageI` implementation over a generated store
  (`9bb2e776`).
- **S4** — an event-sourced account ledger under `cqrsexample` (`40066fff`).

Cached state-view reads with explicit staleness controls landed in `de98d46f`,
and the package moved under the `storage/` grouping in `e512d586`. Two rounds of
external API review followed (`34145e80` additive, `79119556` breaking).

Downstream, keelson adopted generated record stores for durable facts
([ADR-0105](../adr/0105-keelson-adopts-generated-record-stores.md), accepted),
and `DimensionStore` was built on top for interned facts and provenance stamps
([ADR-0112](../adr/0112-dimensionstore-interned-facts-additive-memberships.md),
**proposed**).

## leeway: nested-struct marshalling

The marshaller gained a front end that consumes nested Go structs rather than
only flat ones, across three slices:

- **Slice A** — reflect front-end (`3c800dbb`), single containers and the `S=0`
  splice (`8d77678f`), `Optional` and static-`Many` cardinality (`be638b9d`),
  the `lw` marker package (`584c6ed3`), dynamic `lw.*` memberships (`cd1d406b`).
- **Slice B** — named value-struct `Add` API on generated sections (`3e89eb50`).
- **Slice C** — nested-section codegen for cardinality and static membership
  (`26827691`), then dynamic membership markers (`0b294be9`).

Four ADRs sequence the membership work:
[ADR-0101](../adr/0101-leeway-marshall-mixed-shape-sections.md) mixed-shape
multi-sub-column sections (accepted),
[ADR-0103](../adr/0103-leeway-marshall-dynamic-membership-tuples.md)
dynamic-membership tuples (accepted),
[ADR-0109](../adr/0109-leeway-marshall-multi-membership-ref-tuples.md)
multi-membership and ref-channel tuples (accepted), and
[ADR-0113](../adr/0113-leeway-marshall-nested-primary-consolidation.md)
nested-primary consolidation (**proposed**). ADR-0110 was folded into ADR-0113
rather than kept (`634e62f1`).

Network types landed end-to-end over the same period: IPv4 hosts as Arrow
`uint32` (`c593b04e`), type-safe `lw.IPv4Prefix` / `lw.IPv6Prefix` CIDR lanes
(`00687911`), `netip.Addr` and `netip.Prefix` read accessors (`13225239`,
`b8310ec1`), and an Arrow round-trip test (`adb44d9d`).

Also: a ClickHouse table-clause seam for leeway DDL
([ADR-0102](../adr/0102-leeway-clickhouse-table-clause-seam.md), accepted), and
`TableDesc` containment via `IsSubset` / `Relate` (`79710d82`).

## play: the reactive query workbench

Most of the new user-facing surface landed here.
[ADR-0097](../adr/0097-play-reactive-query-graph.md) completed slices 5 and 6.

**Slice 5 — the signal store.** A signal-store substrate (`be7a91ea`) into which
selection became a real signal, retiring the strangler paths (`43c75d4f`).
Bands extent now rides `{tl_min}` / `{tl_max}` signals (`a33a2780`), the map
raster rides the `vp_*` param seam (`1d45d69e`, realizing
[ADR-0096](../adr/0096-play-geo-raster-map-panel.md) SD6), and signal-writing
widgets plus a main live toggle closed the slice (`84869b75`).

**Slice 6 — first-class tabs.** Every dock tab is a registered `TabSpec`
(`4398a95c`), the registry's `Specs` became the embedder read surface
(`9dbd0cc9`), and per-panel node binding brought selection coherence
(`b22a033e`).

New result panes over the active result:

- **World** — a country choropleth
  ([ADR-0114](../adr/0114-play-world-choropleth-panel.md), accepted), with the
  schematic [`worldmap`](../../public/thestack/imzero2/egui2/widgets/worldmap)
  widget beneath it (`71f9c157`, `efa15d36`).
- **Kanban** — a board from a query result, with the lane inventory read from a
  `lanes` CTE ([ADR-0122](../adr/0122-play-kanban-panel.md), accepted;
  `536e4bad`, `fc1a2cf3`). This pane subsumed the `adrboard` app.
- **Schema** — inspects the leeway schema inferred from results (`10af959f`),
  later reconstructing the real schema (`a21237b8`).
- **Diagnostics** — a dedicated error pane with an EXPLAIN AST endpoint probe
  (`831c9527`).
- **System graph** — the reactive surface rendered as a layered drawing
  (`19f2f2ff`).
- **Map** — a local ADS-B demo loader (`1e7ebe99`), hourly chunked loading
  (`0c9ade5a`), and pluggable raster render (`6366647a`).

Other play work: friendly leeway column handles
([ADR-0116](../adr/0116-play-leeway-column-handle-resolution.md), **proposed**;
`cfad71da`, `854589ef`), content-typed detail cells spelled `label@mime`
([ADR-0123](../adr/0123-play-content-typed-detail-cells.md), **proposed**;
`edafc3fa`), parameter-editing widgets with stem pairing for ranges
([ADR-0124](../adr/0124-play-param-editing-widgets.md), accepted; `3352d19b`),
and the `SPINNAKER_PLAY_*` environment variables renamed to `BOXER_PLAY_*`
(`e8f1b0bc`).

## Query triage and provenance

Three additions answer, for a given query or binary, what it is actually doing:

- **extbin** ([ADR-0118](../adr/0118-extbin-external-process-chokepoint.md),
  **proposed**) — a single audited chokepoint for external-binary *resolution*
  at [`public/extbin`](../../public/extbin) (`7d30dc0e`). Remaining `os/exec`
  sites were migrated onto the registry and the boundary enforced with CS012
  (`2df3c172`); the registry is exposed as `keelson.extbin` with blake3
  attestation (`20c5ab66`). Note the scope: the chokepoint covers resolution,
  not invocation.
- **Passthrough-table classifier**
  ([ADR-0117](../adr/0117-passthrough-table-classifier.md), accepted;
  `4363ce87`) — detects 1:1 pure-retrieval queries, surfaced in play as an
  information-retrieval badge (`eb033ac8`), and failing closed on duplicate CTE
  names (`10d0372b`).
- **Selection conditions**
  ([ADR-0121](../adr/0121-selection-condition-columns.md), accepted; `b06c780b`)
  — decomposes a retrieval query's `WHERE` into `cond_N` columns reporting which
  part admitted each row.

The package-capability story moved the other way. ADR-0120's capability survey
was **withdrawn** (`e42e3b97`) as redundant with the ADR-0026 §SD10 app-scoped
gate, and the `capsurvey` package and its capslock vocabulary were removed
(`e297af3a`, `bb86b0a2`). §SD10 was instead rewired to run capslock in-process
and enforce it (`aa21db40`), with `READ_SYSTEM_STATE` split by the function that
incurs it (`57cfdd48`).

## identity

[ADR-0106](../adr/0106-identity-fibonacci-tags-build-tag-retirement.md)
(accepted) made fibonacci-coded tags the only scheme across six slices: the
codec and tag-value advisor hardened (`a6d24872`), fixed-width tags replaced and
the build-tag axis retired (`89c2e375`), generator fixes for factory exclusivity
and chunked batch commits (`36ea52b0`), an `LW_ID_*` ClickHouse macro pass with
UDF emission and a Go/SQL golden lock (`17f28e46`), a tag-planning API replacing
the stats table (`996ebbc2`), and `identcontainer` / `vlq` culled (`3639010b`).
`leeway id udf` emits the `CREATE FUNCTION` statements (`3d016197`).

Separately,
[ADR-0111](../adr/0111-identity-technology-neutral-leased-id-generation.md)
(**proposed**) made leased id generation technology-neutral (`2a2106c2`), with
`context.Context` threaded through the generator seam (`8e865f95`).

## caching

Versioned admission, dirty-window pins and a freshness TTL (`b78973a5`), SIEVE
victim selection for L1 (`d4098417`), and `S3FIFOStash`, an S3-FIFO-derived
victim cache (`acc1976d`). The eviction-policy choice was settled by an executed
measured gate (`46c9104a`).

The verification work is unusual enough to call out: a Quint model of the
versioned write-through cache (`5a73bf7e`), a replay-liveness TLC pair with
recorded Apalache proof status (`d8f261e1`), and ITF spec traces replayed
against the code (`0ead4acb`), alongside a fuzz corpus and a model oracle
(`a899e22e`). [ADR-0104](../adr/0104-caching-post-review-hardening.md) remains
**proposed**.

## SQL and nanopass

`WITH RECURSIVE` landed end-to-end: parsed in both grammars (`7c0c0300`),
carried through the AST and all emitters (`19309480`), recognized in scope
analysis (`191a1dcb`), and surfaced in play as recursive CTE nodes (`704d2023`,
realizing ADR-0097 SD9). Table functions accept expression arguments, including
`values()` row tuples (`eed29f98`).

The keelson SQL pass registry
([ADR-0108](../adr/0108-keelson-sql-pass-registry.md), accepted) added a
pre-execute stage with an introspectable catalog at
[`public/keelson/data/passreg`](../../public/keelson/data/passreg) (`a573c272`),
applied at both SQL executors (`811e0770`), with late-bound `Factory` entries
per §SD7 (`81daf32d`). play shows the statement as sent to the server
(`7aae711e`).

# How to migrate downstream code after the May 2026 quality pass

This recipe covers the breaking surface introduced by commits `cc1e19b…fbcf61d`
on `main`. Internal-only changes (redundant `break` removal, dead-code
deletion, receiver renames, simplifications) need no migration steps and are
omitted; the table in [§When to use this recipe](#when-to-use-this-recipe)
flags which buckets are breaking.

## When to use this recipe

You depend on a `github.com/stergiotis/boxer` revision older than `cc1e19b` and
the upgrade fails to compile or behaves differently. The pass touched eight
named buckets; only the rows marked **yes** below require code changes
downstream.

| Bucket | Commit | Break? | Action needed downstream? |
|---|---|---|---|
| A — redundant `break` removal | `cc1e19b` | no | none |
| F — swallowed errors + EncodeFloat32 bug fix | `990879b` | behavior change | re-verify CBOR golden tests; see [§Step 5](#step-5-cborencoderencodefloat32-now-writes-the-spec-correct-byte-count) |
| H — staticcheck simplifications | `a78affd` | no | none |
| D — deprecated API migration | `d6e1812` | soft | `UnsafeStringToByte` still works via a `Deprecated:` alias; sweep when convenient, see [§Step 4](#step-4-rename-unsafestringtobyte-to-unsafestringtobytes-optional) |
| C — duplicate / architecture | `800f2f0` | **yes** | three small renames, see [§Step 1](#step-1-bucket-c--small-renames) |
| B — U1000 dead-code deletion | `32cb82e` | none in practice | the symbols had zero callers anywhere in the repo when deleted |
| E1 — short-decl in `PanicHandler` | `668eb7c` | no | none |
| E2 — receiver names → `inst` | `bbde5f7` | no | none |
| E3 — interface `I` suffix | `aa7771c` | **yes** | rename 8 identifiers, see [§Step 2](#step-2-interfaces-gain-the-i-suffix) |
| E4 — enum `E` suffix | `fbcf61d` | **yes** | rename `Tristate*`, see [§Step 3](#step-3-tristate-gains-the-e-suffix) |

## Prerequisites

- A working tree with all your boxer-using code committed (the recipe edits
  source files in-place; a clean tree makes the diff legible).
- GNU `sed` (the snippets use the in-place `-i` form without a backup
  argument; macOS `sed` users should substitute `-i ''`).
- A successful `go build ./...` before you start, so failures introduced by
  the recipe are easy to spot.

## Steps

### Step 1: Bucket C — small renames

**1a. `golay24.DecodeLowLevel2` was deleted.** It was byte-for-byte identical
to [`github.com/stergiotis/boxer/public/fec/code/golay24.DecodeLowLevel`](https://pkg.go.dev/github.com/stergiotis/boxer/public/fec/code/golay24#DecodeLowLevel)
plus an unused `nCodeWords` parameter; no callers existed anywhere in the
repo when it was removed.

```diff
- golay24.DecodeLowLevel2(dst, src, n)
+ golay24.DecodeLowLevel(dst, src)
```

`DecodeLowLevel0` and `DecodeLowLevel1` are distinct algorithms and remain
in place.

**1b. `vsc.go` → `vcs.go` in [`github.com/stergiotis/boxer/public/observability/vcs`](https://pkg.go.dev/github.com/stergiotis/boxer/public/observability/vcs).**
Pure file rename, package and API unchanged. No code action required; the
next `git pull` does it for you.

**1c. `common.IntermediateColumnIterator` is now a true named type, not a
type alias** (CODINGSTANDARDS bans aliases). The underlying type is
unchanged but assignability between distinct named types now requires a
conversion.

If you wrote function signatures that spelled out the underlying iter,
switch to the named type:

```diff
- func LoadFromIntermediates(it iter.Seq2[common.IntermediateColumnContext, *common.IntermediateColumnProps]) error
+ func LoadFromIntermediates(it common.IntermediateColumnIterator) error
```

If you must pass a raw `iter.Seq2[…]`, convert at the boundary:

```go
LoadFromIntermediates(common.IntermediateColumnIterator(myIter))
```

### Step 2: Interfaces gain the `I` suffix

Eight interfaces were renamed to comply with CODINGSTANDARDS' interface
naming rule. Methods, signatures, and semantics are unchanged — only the
type names move.

| Old name | New name | Package |
|---|---|---|
| `Configer` | `ConfigerI` | [`github.com/stergiotis/boxer/public/config`](https://pkg.go.dev/github.com/stergiotis/boxer/public/config) |
| `ByteBlockWriter` | `ByteBlockWriterI` | [`github.com/stergiotis/boxer/public/ea`](https://pkg.go.dev/github.com/stergiotis/boxer/public/ea) |
| `ByteReadReader` | `ByteReadReaderI` | [`github.com/stergiotis/boxer/public/ea`](https://pkg.go.dev/github.com/stergiotis/boxer/public/ea) |
| `ByteBlockDiscardReader` | `ByteBlockDiscardReaderI` | [`github.com/stergiotis/boxer/public/ea`](https://pkg.go.dev/github.com/stergiotis/boxer/public/ea) |
| `MessageReader` | `MessageReaderI` | [`github.com/stergiotis/boxer/public/fec/ea`](https://pkg.go.dev/github.com/stergiotis/boxer/public/fec/ea) |
| `MessageWriter` | `MessageWriterI` | [`github.com/stergiotis/boxer/public/fec/ea`](https://pkg.go.dev/github.com/stergiotis/boxer/public/fec/ea) |
| `ErrorWithStructuredData` | `ErrorWithStructuredDataI` | [`github.com/stergiotis/boxer/public/observability/eh`](https://pkg.go.dev/github.com/stergiotis/boxer/public/observability/eh) |
| `EncoderWriter` | `EncoderWriterI` | [`github.com/stergiotis/boxer/public/semistructured/cbor`](https://pkg.go.dev/github.com/stergiotis/boxer/public/semistructured/cbor) |

CODINGSTANDARDS forbids type aliases, so no backward-compat shim is left
behind: downstream code must rename verbatim.

**Bulk migration recipe.** From your consumer repo's root:

```bash
ren() { find . -name '*.go' -not -path './vendor/*' -exec sed -i "s/\\b$1\\b/$2/g" {} +; }
ren Configer ConfigerI
ren ByteBlockWriter ByteBlockWriterI
ren ByteReadReader ByteReadReaderI
ren ByteBlockDiscardReader ByteBlockDiscardReaderI
ren MessageReader MessageReaderI
ren MessageWriter MessageWriterI
ren ErrorWithStructuredData ErrorWithStructuredDataI
ren EncoderWriter EncoderWriterI
go build ./...
```

Word-boundary `\b` is load-bearing: `ByteReadReader` is a prefix of
`ByteReadReaderI`, so a careless replace would corrupt already-migrated
lines on a second run.

### Step 3: `Tristate` gains the `E` suffix

[`github.com/stergiotis/boxer/public/logical`](https://pkg.go.dev/github.com/stergiotis/boxer/public/logical)
gained the standard's enum suffix. The type, its three values, and the
helper function all change name:

| Old | New |
|---|---|
| `logical.Tristate` | `logical.TristateE` |
| `logical.TriFalse` | `logical.TristateFalse` |
| `logical.TriNil` | `logical.TristateNil` |
| `logical.TriTrue` | `logical.TristateTrue` |
| `logical.TriFromBool(b)` | `logical.TristateFromBool(b)` |

```bash
ren() { find . -name '*.go' -not -path './vendor/*' -exec sed -i "s/\\b$1\\b/$2/g" {} +; }
# Order matters: rename the type last, otherwise the value renames stop
# matching once `Tristate` has become `TristateE`.
ren TriFromBool TristateFromBool
ren TriFalse   TristateFalse
ren TriNil     TristateNil
ren TriTrue    TristateTrue
ren Tristate   TristateE
```

[`github.com/stergiotis/boxer/public/semistructured/cbor`](https://pkg.go.dev/github.com/stergiotis/boxer/public/semistructured/cbor)
also gained `cbor.MajorType` → `cbor.MajorTypeE`, but the `MajorType*`
constants were already correctly prefixed and need no change. Sweep
references with `ren MajorType MajorTypeE` if you spell the type out
directly — most consumers use the higher-level encoder methods
(`EncodeUint`, `EncodeString`, …) and never name the type.

### Step 4: rename `UnsafeStringToByte` to `UnsafeStringToBytes` (optional)

The old name continues to work as a `Deprecated:` alias forwarding to
`UnsafeStringToBytes` in [`github.com/stergiotis/boxer/public/unsafeperf`](https://pkg.go.dev/github.com/stergiotis/boxer/public/unsafeperf).
No migration is strictly required, but `staticcheck` SA1019 will flag
remaining callers. To clear them:

```bash
ren() { find . -name '*.go' -not -path './vendor/*' -exec sed -i "s/\\bUnsafeStringToByte\\b/UnsafeStringToBytes/g" {} +; }
ren
```

### Step 5: `cbor.Encoder.EncodeFloat32` now writes the spec-correct byte count

Prior to commit `990879b`, [`(*cbor.Encoder).EncodeFloat32`](https://pkg.go.dev/github.com/stergiotis/boxer/public/semistructured/cbor#Encoder.EncodeFloat32)
wrote **8 bytes** after the type header instead of the spec-correct 4 bytes
(a slice-header bug in which `binary.BigEndian.AppendUint32`'s return value
was discarded). Any persisted CBOR produced by `EncodeFloat32` before the
fix is malformed: 4 spurious trailing zero bytes overlap whatever item
came next.

If you hold such artifacts:

1. They were already malformed; most CBOR decoders would have failed on
   them.
2. There is no automated rewrite — the trailing bytes belong to whatever
   item came after the `float32`, which is context-dependent.
3. Re-encoding from your source data is the safe path.

`EncodeFloat64` was unaffected; only `EncodeFloat32` had this bug.

## Verification

After applying the steps that apply to your repo, run:

```bash
go build ./...
go test ./...
```

Both should succeed. If they do not, see Troubleshooting.

To confirm the sweep is complete, search for the old names. There should be
no hits inside your own code:

```bash
grep -rEn '\b(Configer|ByteBlockWriter|ByteReadReader|ByteBlockDiscardReader|MessageReader|MessageWriter|ErrorWithStructuredData|EncoderWriter|Tristate|TriFalse|TriNil|TriTrue|TriFromBool|DecodeLowLevel2|UnsafeStringToByte)\b' --include='*.go' . | grep -v vendor/
```

A worked example of the sweep applied to a downstream consumer
(`pebble2impl`) lives at commit `1e81f930` in that repository; the diff
covers 5 sites across 4 packages.

## Troubleshooting

- **Symptom:** `undefined: config.Configer` (or any of the renamed types).
  **Cause:** the sweep missed a call-site, or the second run of `ren` did
  not find the old name because the first run already converted it.
  **Fix:** rerun `go build ./...` to get the exact file and line, then
  rename that occurrence by hand.

- **Symptom:** the `ren` helper turns `ByteReadReaderI` into
  `ByteReadReaderII`.
  **Cause:** the `\b` word-boundary anchor was dropped — for example by
  copy-pasting a snippet that pre-expanded the backslash. The double-suffix
  is a tell that the sed pattern matched a prefix instead of a whole word.
  **Fix:** revert the over-renamed file from git, restore the `\b\b`
  anchors in your `ren` invocation, and re-run.

- **Symptom:** `cannot use myIter (variable of type iter.Seq2[…]) as
  common.IntermediateColumnIterator value`.
  **Cause:** the type-alias-to-named-type change (Step 1c). The underlying
  types are identical but Go's assignability rules require a conversion
  between distinct named types.
  **Fix:** add an explicit conversion: `common.IntermediateColumnIterator(myIter)`.
  Alternatively, change the producing signature to return
  `common.IntermediateColumnIterator` directly.

- **Symptom:** CBOR round-trip tests fail after upgrading, only for
  payloads containing `float32` values.
  **Cause:** Step 5 — your golden fixtures were produced by the buggy
  `EncodeFloat32` and contain 4 spurious trailing zero bytes per
  `float32` value.
  **Fix:** regenerate the fixtures from source data with the fixed
  encoder. Verify by decoding the regenerated fixture with an independent
  CBOR implementation (e.g. `cbor.Diagnose`).
